Leniwa ewaluacja

Czy pamiętasz mój program obliczający kwadraty liczb całkowitych w Clojure? 
(take 25 (squares-of (integers)))
Czy pamiętasz jak pokazywałem Ci jak ten program wygląda w Javie?
public class Squint {
  public  static  void  main(String  args[])  {
    for  (int  i=1;  i<=25;  i++)
      System.out.println(i*i);}
}
No cóż, nie było to do końca dobre porównanie. Tutaj o wiele bardziej sprawiedliwe porównanie programu napisanego w Javie.
take(25, squaresOf(integers()));
Zauważ, że jedyne, co zrobiłem to przesunąłem nawiasy, dodałem przecinek i użyłem nazw pasujących do konwencji.
Oto nieco szerszy kontekst:
@Test
public void squaresOfIntegers() throws Exception {
  assertEquals(asList(1, 4, 9, 16, 25), squint(5));
}

private List<Integer> squint(int n) {
  return take(n, squaresOf(integers()));
}
Czy potrafisz napisać taki program w Javie? Dam Ci wskazówkę: trzeba użyć leniwej ewaluacji tak samo jak w programie w Clojure. O to chodzi.
